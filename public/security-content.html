SECURITY STATEMENT

Last Updated: February 28, 2026

VTAC MANAGER is designed with layered security controls to protect customer data and service availability.

Infrastructure

VTAC MANAGER is hosted on trusted cloud providers, including Vercel and Supabase (AWS), and uses provider-managed security controls for infrastructure operations.

Encryption

  • Data in transit is protected with TLS 1.2+.
  • Data at rest is encrypted using provider-managed encryption standards (AES-256).

Authentication & Access Control

  • User authentication is managed by Supabase Auth.
  • Passwords are hashed and not stored in plain text.
  • Database-level access controls (including Row-Level Security) are used to isolate team data.

Application Security

We continuously improve security through dependency patching, secure development practices, and access-control hardening.

Backups & Reliability

We maintain regular backup and recovery procedures to support service continuity.

Payments

VTAC does not store raw payment card details on application servers. Payments are processed by PCI-compliant providers (currently Stripe in production flows).

Incident Response

VTAC maintains an internal incident response process for detection, containment, remediation, and post-incident review. Notifications are handled according to applicable legal requirements.

Vulnerability Reporting

If you believe you found a security issue, please report it to: security@vtacmanager.com
